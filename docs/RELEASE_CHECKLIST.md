## 1. 范围与版本

- [ ] 发布范围、目标用户和明确不包含的能力已写入发布说明。
- [ ] `package.json` 版本、产品名称、App ID 和产物名称正确。
- [ ] 当前提交已评审并具有可恢复标签；工作树无未提交改动。
- [ ] 数据结构或 Launcher Session 发生变化时，已提供迁移、兼容和回滚说明。
- [ ] UE5 团队已确认所使用的 `contractVersion`。
- [ ] 用户可见的重要变化、限制和数据影响已记录。

## 2. 干净构建

在新的仓库副本或清理后的 CI 工作区执行：

```powershell
pnpm install --frozen-lockfile
pnpm check
pnpm audit --prod --audit-level high
pnpm dist:win
```

- [ ] 安装严格使用锁文件，未临时修改依赖或跳过安装脚本错误。
- [ ] React 与 Electron 类型检查通过。
- [ ] 账户、AI、契约和资产测试全部通过。
- [ ] Vite/Electron 生产构建通过，`dist` 中无旧哈希残留。
- [ ] 生产依赖无未处置的高危或严重漏洞。
- [ ] Windows portable 产物从上述同一提交生成。

任何测试被跳过、审计例外或构建警告都必须有负责人、原因、影响、到期日和跟踪问题。

## 3. 密钥与隐私

- [ ] 仓库当前内容和 Git 历史已用秘密扫描工具检查。
- [ ] `dist`、`dist-electron`、`release`、压缩备份和安装包已检查。
- [ ] 产物不包含真实 API Key、密码、令牌、私钥或内部服务凭据。
- [ ] 产物不包含 `accounts.json*`、`ai.json`、游戏会话文件、聊天记录或测试用户资料。
- [ ] 示例配置只包含明显占位符，没有开发者本机绝对路径。
- [ ] 任何曾出现在聊天、截图、日志或历史提交中的密钥已经撤销并轮换。
- [ ] 本次版本收集、发送和保留的个人数据已完成产品与隐私审核。
- [ ] AI 请求字段符合最小化原则，供应商和删除/保留方式已披露。
- [ ] 日志和崩溃报告已确认不会记录邮箱、完整人格提示、聊天正文、ticket 或密钥。

## 4. 自动化与模块验收

- [ ] 并发注册不会丢失账户，邮箱唯一约束行为符合预期。
- [ ] 不存在的日历日期和未来日期被拒绝。
- [ ] 首次保存后，所有已填写字段不能通过 UI 或 IPC 被覆盖。
- [ ] 原本为空的字段可以补充，爱好槽位限制正确。
- [ ] 每账户 Agent ID 唯一，Agent 档案独立，模型策略明确为共享基础模型。
- [ ] 旧账户 Agent 策略能够兼容迁移，不破坏登录。
- [ ] provider、Base URL、API 路径和模型选择兼容。
- [ ] AI 未配置、超时、429、5xx、空响应和取消时均给出安全且可理解的错误。
- [ ] Launcher Session 缺失 `agent` 时契约测试失败，合法 `1.0` 示例通过。
- [ ] GLB 大小、格式、网格、纹理、骨骼和预览图通过资产门禁。

## 5. Windows 桌面冒烟测试

至少在一台没有源码、Node.js 和开发配置的测试机上执行。

- [ ] 双击 portable EXE 可以启动，未依赖项目目录或 `node_modules`。
- [ ] 只启动一个应用实例；第二次启动会恢复已有窗口。
- [ ] 最小化、最大化/还原和关闭按钮正确。
- [ ] 1120×720 最小窗口和 100%/125%/150% 缩放下文字可读，关键控件不重叠。
- [ ] 断网、模型服务不可用和资源加载失败不会出现白屏或无法退出。
- [ ] 开发者工具、任意外部导航、新窗口、WebView 和未授权权限不能绕过限制。
- [ ] 启动、退出和卸载后没有遗留不应存在的后台进程。

## 6. 用户流程

使用专用测试 Windows 用户或隔离的 `userData`，不要删除开发者真实账户数据。

- [ ] 新用户完成注册后进入自然的首次资料流程，而不是直接进入主页。
- [ ] 非法邮箱、短密码、重复邮箱和错误密码提示正确。
- [ ] 登录成功前无法访问主界面、AI、伴生模式或游戏启动。
- [ ] 资料保存后出现 2–3 秒启动过渡，再进入主界面。
- [ ] 强调色在资料流程、主页、聊天和重启后保持一致。
- [ ] 世界、社交、虚拟分身和个人主页可进入，填充内容不会造成误导性真实承诺。
- [ ] 退出登录位于个人主页，退出后敏感页面和当前会话不可继续访问。
- [ ] 应用重启后，已保存资料仍存在且不可覆盖。
- [ ] 账户文件损坏演练能够回退有效备份，双重损坏时给出可恢复的错误说明。

## 7. AI 与虚拟分身

- [ ] 新账户获得独立 Agent 档案，但文案不声称独占模型权重或实例。
- [ ] 未完成资料时 Agent 保持 `WAITING_FOR_PROFILE`，完成后变为 `READY`。
- [ ] 人格回复使用已知资料，未知字段不会被虚构为玩家事实。
- [ ] 提示注入不能诱导 AI 暴露系统提示、密钥或改变身份边界。
- [ ] 清空对话时，迟到的旧请求不会重新写回已清空会话。
- [ ] 连续点击发送不会产生无限并发或重复计费。
- [ ] 3D 分身主页面加载、缩放和旋转正常；失败时有静态降级而非白屏。
- [ ] 页面隐藏时渲染暂停或降频，伴生模式帧率和资源占用符合预算。
- [ ] 进入伴生模式后主窗口隐藏，透明背景、拖动、双击/按钮恢复和退出行为正确。
- [ ] 多次进入/退出伴生模式后无持续增长的 GPU/内存占用。

## 8. UE5 联调

- [ ] 未配置或路径错误时，Launcher 给出明确错误且不遗留会话文件。
- [ ] 带空格和中文的 UE5 路径能够正确启动。
- [ ] UE5 能读取契约版本、Launch ID 和会话文件路径。
- [ ] 会话包含匹配的账户、Agent、语言和渠道信息。
- [ ] UE5 对缺失、损坏、版本不支持和过期会话给出可诊断错误。
- [ ] 启动失败、游戏退出或会话过期后文件被清理。
- [ ] Development 构建可按约定接受 `prototype-local`；Shipping 构建明确拒绝。
- [ ] 正式版一次性 ticket 已验证不可重放，并通过 HTTPS 后端交换。

## 生产发布额外阻断条件

以下项目未完成时，只能发布标记清晰的内部原型，不能发布生产版：

- [ ] 已用服务端账户与认证替换本地 JSON 账户来源。
- [ ] 已用服务端 AI 网关替换客户端供应商密钥。
- [ ] 已实现短时一次性游戏 ticket 的服务端签发、交换和撤销。
- [ ] 已完成隐私政策、同意、导出、删除、保留和供应商数据处理流程。
- [ ] 已建立限流、风控、脱敏审计、崩溃报告、监控和告警。
- [ ] 已完成 E2E、安全测试、代码签名、更新验证和灾难恢复演练。
